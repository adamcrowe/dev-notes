Bootstrap 4 Tooltips

The Tooltip is small pop-up box that appears when the user moves the mouse pointer over an element

Overview

Tooltips rely on the 3rd party library Popper.js for positioning. You must include popper.min.js before bootstrap.js or use bootstrap.bundle.min.js / bootstrap.bundle.js which contains Popper.js in order for tooltips to work!

Note: Tooltips are opt-in for performance reasons, so you must initialize them yourself.Tooltips must be initialized with jQuery: select the specified element and call the tooltip() method.

The following code will enable all tooltips in the document:

<script>
$(document).ready(function(){
  $('[data-toggle="tooltip"]').tooltip();
});
</script>

The following code will enable a tooltip on the specified element:

<script>
$(document).ready(function(){
  $('#myTooltip').tooltip();
});
</script>

The following code will enable a tooltip on the specified selector:

<script>
$(document).ready(function(){
  $('body').tooltip({title: "Hooray!", selector: ".btn"});
});
</script>

Basic Tooltip

Add the data-toggle="tooltip" attribute to an element.

Use the title attribute to specify the text that should be displayed inside the tooltip:

Hover over me

Title

The title option (default: "") specifies the text that should be displayed inside the tooltip.

Trigger

The trigger option (default: "hover focus") specifies how the tooltip is triggered.

Possible values (To pass multiple values, separate them with a space):

  • "click" - Trigger the tooltip with a click
  • "hover" - Trigger the tooltip on hover
  • "focus" - Trigger the tooltip when it gets focus (by tabbing or clicking .e.g
  • "manual" - Trigger the tooltip manually

Positioning

The placement option (default: "top") specifies the tooltip position.

  • Top
  • Bottom
  • Left
  • Right

Tip: You can also use the data-placement attribute with a value of "auto", which will let the browser decide the position of the tooltip. For example, if the value is "auto left", the tooltip will display on the left side when possible, otherwise on the right.

Disabled

Animation

The animation option (default: true) specifies whether to add a CSS fade transition effect to the tooltip.

Container

The container option (default: false) appends the tooltip to a specific element.

Delay

The delay option (default: 0) specifies the number of milliseconds it will take to show and hide the popover.

This tooltip will take 1000 milliseconds to show when you hover over it, and 1000 milliseconds to hide on mouse leave.



This tooltip will take 500 milliseconds to show when you hover over it, and 100 ms to hide on mouse leave.

HTML

The html option (default: false) specifies whether to accept HTML tags in the tooltip.

When set to false (default), jQuery's text() method will be used. Use this if you are worried about XSS attacks.

Template

Base HTML to use when creating the tooltip. The tooltip's title will be inserted into the element having the class .tooltip-inner and the element with the class .tooltip-arrow will become the tooltip's arrow. The outermost wrapper element should have the .tooltip class.

<div class="tooltip" role="tooltip">
  <div class="tooltip-arrow"></div>
  <div class="tooltip-inner"></div>
</div>

Viewport

Keeps the tooltip within the bounds of this element (default: {selector: "body", padding: 0}).

Example: viewport: '#viewport' or {selector: '#viewport', padding: 0}

If a function is given, it is called with the triggering element DOM node as its only argument. The this context is set to the tooltip instance.

Tooltip Options and Methods

See: https://getbootstrap.com/docs/4.0/components/tooltips/

Tooltip will show here

  • .tooltip('show') Shows the tooltip
  • .tooltip('hide') Hides the tooltip
  • .tooltip('toggle') Toggles the tooltip
  • .tooltip('destroy') Hides and destroys the tooltip

Tooltip Events

The show.bs.tooltip event occurs when the tooltip is about to be shown.

The shown.bs.tooltip event occurs when the tooltip is fully shown.

The hide.bs.tooltip event occurs when the tooltip is about to be hidden.

The hidden.bs.tooltip event occurs when the tooltip is fully hidden.
